Webhook and API status

Integration

Use this page to wire Web Scrobbler into Live listening diary. The endpoint is intentionally lightweight so the webhook can return quickly and keep the dashboard live without background jobs or filesystem storage.

POST /api/webhook Protected by secret token JSON ingest + PostgreSQL

Webhook Endpoint

Send a JSON POST request to /api/webhook with one of these auth methods:

  • X-Recently-Played-Token: <WEBHOOK_SECRET>
  • Authorization: Bearer <WEBHOOK_SECRET>
  • POST /api/webhook/<WEBHOOK_SECRET>

Supported event types: nowplaying, paused, resumedplaying, scrobble, loved.

API Routes

  • /api/recent - latest events JSON
  • /api/now-playing - current display card JSON
  • /api/status - summary and route info
  • /history - full playback timeline

The webhook stores only the useful display fields and prunes old rows so the app stays lightweight on Hobby.

Example Payload

{
  "event": "scrobble",
  "song": {
    "artist": "Carly Rae Jepsen",
    "track": "Run Away With Me",
    "album": "Emotion",
    "artwork": "https://example.com/art.jpg",
    "url": "https://example.com/track",
    "timestamp": "2026-04-21T18:14:00Z",
    "loved": false
  }
}

Response Shape

{
  "ok": true,
  "deduped": false,
  "stored_event": {
    "artist": "...",
    "track": "...",
    "received_at": "..."
  }
}
